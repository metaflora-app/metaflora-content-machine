# REPURPOSE
## Один кусок контента, несколько платформенных форматов

Механическое копирование текста под разные платформы не работает, потому что каждый формат устроен по своей внутренней логике: видео держится на темпе речи и монтажной склейке, карусель держится на том, что каждый слайд читается за долю секунды пролистывания, имейл держится на повествовательном разворачивании мысли. Перенос готового куска один в один в другой формат сохраняет слова, но теряет структуру, ради которой формат вообще работает.

Задача этого скилла — сначала правильно извлечь суть исходника в зависимости от его типа, и только потом заново собрать её под правила каждого целевого формата.

---

## ЧТО СКИЛЛ ПРИНИМАЕТ НА ВХОДЕ

Обязательно:
- **готовый исходник**: законченный сценарий карусели, законченный сценарий рилса, войс-нота или транскрипт, либо сырые тезисы/идея одной строкой

Опционально:
- **целевые форматы**: если не указаны, скилл сам предлагает 3-4 наиболее подходящих под тип исходника и объясняет выбор одной строкой на формат
- **voice-профиль** из `voice-print`, если он есть в контексте: применяется одинаково ко всем целевым форматам
- **готовый хук** из `hook-lab`, если он уже есть: используется как якорь для всех форматов вместо того чтобы придумывать новый хук под каждый

---

## ДЕЙСТВИЕ ИЗВЛЕЧЕНИЯ ПО ТИПУ ИСХОДНИКА

| Тип исходника | Что делать перед раскладкой по форматам |
|---|---|
| Готовый сценарий карусели | Извлечь основную мысль и ключевые уроки, отбросить слайд-структуру целиком |
| Готовый сценарий рилса/тиктока | Извлечь хук и структуру аргументации, отбросить темп и монтажные пометки |
| Войс-нота или транскрипт | Сначала очистить от слов-паразитов и оговорок, затем определить одну центральную мысль |
| Сырые тезисы или идея одной строкой | Сначала развить идею до полноценной мысли с примером, только потом раскладывать по форматам |

Пропуск этого шага и прямой перенос исходного текста в целевой формат — самая частая причина, почему переупакованный контент читается как черновик.

---

## КАК ФОРМАТ МЕНЯЕТ СТРУКТУРУ

**Сценарий короткого видео**: хук в первые 0-3 секунды, тело с одной мыслью на монтажную склейку, конкретный CTA. Если структура уже есть в `script-smith`, использовать её логику вместо изобретения новой.

**Карусель**: первый слайд — хук крупным шрифтом, каждый следующий слайд — ровно одна мысль, последний слайд — призыв к действию. Нельзя переносить видео-хук на первый слайд без адаптации: то, что работает на слух, не всегда работает как текст на статичной картинке.

**Отдельный пост/подпись**: держится сам по себе без опоры на визуальный ряд или темп речи. Первая строка выполняет ту же работу что хук в видео, но должна работать чисто текстом.

**Имейл-рассылка**: тема письма плюс повествовательное разворачивание мысли, здесь допустим более медленный темп и больше контекста, чем в коротком видео.

**Тред/X**: первый твит — весь смысл хука, дальше по одной мысли на твит, последний твит — призыв к действию или ссылка.

**Pinterest-пин**: заголовок и короткое описание должны работать как самостоятельная единица, без опоры на остальной контент рядом.

## ПРОЦЕСС

1. Определить тип исходника и применить нужное действие извлечения из таблицы выше.
2. Определить целевые форматы: указанные пользователем, либо 3-4 предложенных под тип исходника.
3. Для каждого формата пересобрать структуру заново по правилам этого формата, не копировать текст исходника построчно.
4. Прогнать самопроверку по чек-листу ниже на каждый формат отдельно.
5. Выдать результат по форматам, с одной строкой объяснения почему выбран именно такой формат под эту тему, если форматы не были явно указаны пользователем.

---

## ЧЕК-ЛИСТ САМОПРОВЕРКИ (перед выдачей)

- каждый формат пересобран по своей структуре, не является текстом исходника с изменёнными переносами строк
- первая строка/первый слайд/первый твит работает как самостоятельный хук именно для этого формата, не дословный перенос хука из другого формата
- длина ориентирована на норму платформы, не на длину исходника
- если voice-профиль есть, тон одинаково узнаваем во всех форматах
- CTA в каждом формате соответствует тому, что реально можно сделать на этой платформе

---

## EDGE CASES

**Исходник слишком короткий для полноценной репёрпозы** (одно предложение, короткая мысль без развития)
→ Не растягивать искусственно. Сказать прямо, что часть форматов не подойдёт при таком объёме материала, предложить сначала развить идею, и repurpose-нуть только те форматы, которые реально держатся на коротком материале.

**Исходник узкоспециализированный, и сжатие теряет смысл**
→ Для форматов с жёстким лимитом длины (тред, Pinterest) явно сказать, что сжатие убирает важный нюанс, и предложить либо смягчённую версию с меньшей точностью, либо пропустить этот формат.

**Пользователь просит конкретные форматы, не даёт скиллу выбирать**
→ Уважать выбор, не навязывать дополнительные форматы сверх запрошенных.

**Войс-нота или транскрипт содержит несколько разных мыслей одновременно**
→ Не смешивать их в одну репёрпозу. Разделить на отдельные центральные мысли и предложить пользователю выбрать одну, либо репёрпозить каждую по отдельности при явном запросе.

**Исходник уже написан под конкретную платформу и пользователь просит адаптировать под ту же платформу**
→ Уточнить задачу. Второй вариант для той же площадки считается вариацией. Здесь подходит переписывание с другим углом подачи.

## СВЯЗЬ С ОСТАЛЬНЫМИ СКИЛЛАМИ ПАКЕТА METAFLORA-CONTENT-MACHINE

**script-smith → repurpose**: готовый сценарий видео — один из принимаемых типов исходника напрямую, без дополнительной подготовки.

**hook-lab → repurpose**: если хук уже выбран, он становится якорем для всех целевых форматов, адаптируется под правила каждого формата, но не придумывается заново с нуля для каждого.

**voice-print → repurpose**: голос должен оставаться узнаваемым при смене формата, это одна из главных проверок в чек-листе выше.

Если исходника ещё нет и пользователь хочет контент под несколько платформ с нуля, разумнее сначала собрать один полноценный формат через `script-smith` или `hook-lab`, а затем передать готовый результат сюда.

---

## ЧТО СКИЛЛ НИКОГДА НЕ ДЕЛАЕТ

- не переносит текст исходника в целевой формат без пересборки под его структуру
- не подгоняет все форматы под один и тот же хук без адаптации к особенностям формата
- не растягивает искусственно короткий исходник до полноценных форматов, которые он не тянет по объёму
- не смешивает несколько разных центральных мыслей в одну репёрпозу
- не навязывает дополнительные форматы сверх тех, что явно запросил пользователь
- не жертвует точностью ради того чтобы уместиться в лимит формата без явного предупреждения об этом
