# Metaflora Content Machine

Набор из пяти переносимых навыков для работы с контентом. Он помогает пройти путь от реальных слов аудитории и авторского голоса до сценария, поста, карусели, письма или треда.

Каждый навык работает отдельно. В общей цепочке следующий шаг получает подготовленный материал и может продолжить работу без повторного разбора входа.

![Карта Content Machine](assets/svg/content-machine-map.svg)

## Что внутри

| Навык | Задача | Что получается на выходе |
| --- | --- | --- |
| [`voice-print`](voice-print/SKILL.md) | Снимает портрет авторского голоса с текстовых образцов | `VOICE-PROFILE` с лексикой, ритмом, границами тона и тестовой фразой |
| [`audience-dig`](audience-dig/SKILL.md) | Разбирает комментарии, DM, отзывы и вопросы | `AUDIENCE-INTEL` с повторяющимися фразами, болями, вопросами и опорными хуками |
| [`hook-lab`](hook-lab/SKILL.md) | Собирает первые строки под тему и площадку | Варианты хуков с формулой, объяснением и проверкой на клише |
| [`script-smith`](script-smith/SKILL.md) | Собирает короткий видеосценарий | Таймированный сценарий с хуком, доверием, телом и CTA |
| [`repurpose`](repurpose/SKILL.md) | Пересобирает одну идею для разных площадок | Нативные версии для видео, карусели, поста, письма, треда и Pinterest |

## Рабочая схема

```text
тексты автора ─────> voice-print ─────────────┐
                                               │
комментарии / DM / отзывы ─> audience-dig ─> hook-lab ─> script-smith ─> repurpose
                                               │                               │
                                               └──── голос сохраняется ────────┘
```

### Полный проход

1. Соберите 3–10 связанных текстов одного автора и запустите `voice-print`.
2. Принесите комментарии, переписки или отзывы в `audience-dig`.
3. Передайте найденную боль, вопрос или готовую фразу в `hook-lab`.
4. Выберите хук и разверните его через `script-smith` под нужную длительность.
5. Отдайте готовый сценарий в `repurpose`, чтобы собрать другие площадки без механического копирования.

Пример цепочки в одном рабочем чате:

```text
1. вот восемь моих постов. собери voice-profile
2. вот 60 комментариев под последними роликами. вытащи повторяющиеся боли
3. сделай хуки для Reels на основе фразы «я уже пробовал, но опять бросил»
4. разверни второй вариант в ролик на 45 секунд
5. из этого сценария сделай карусель, пост в Telegram и письмо
```

Подробный разбор потока есть в [документации по процессу](docs/workflows.md).

## С чего начать

### Нужен голос для будущих текстов

Откройте [`voice-print`](voice-print/SKILL.md) и отправьте 3–10 собственных примеров.

```text
вот мои посты за последний месяц. разберись, как я пишу,
и собери переносимый voice-profile для будущих Reels, постов и писем
```

### Есть много комментариев, но нет тем

Откройте [`audience-dig`](audience-dig/SKILL.md) и вставьте сырой массив сообщений.

```text
ниже 48 комментариев под роликами про финансы.
найди формулировки проблем, вопросы и 3 темы, которые стоит разобрать следующими
```

### Нужны первые строки

Откройте [`hook-lab`](hook-lab/SKILL.md).

```text
сделай 7 вариантов первого кадра для Reels про то,
как фрилансеру перестать терять заявки в Telegram.
без выдуманных цифр и без рекламных клише
```

### Нужен сценарий для ролика

Откройте [`script-smith`](script-smith/SKILL.md).

```text
разверни этот хук в сценарий для Reels на 30 секунд.
тема: заявки теряются в личных сообщениях.
пусть текст звучит как живая речь, без канцелярита
```

### Один материал должен пойти в несколько мест

Откройте [`repurpose`](repurpose/SKILL.md).

```text
вот готовый сценарий. пересобери его в карусель на 7 слайдов,
пост в Telegram и короткое письмо в рассылку.
не копируй формулировки строка в строку
```

## Установка

Каждый навык лежит в отдельной папке и содержит один `SKILL.md`. Можно поставить один навык или весь набор.

### Claude.ai

1. Скачайте нужную папку, например `hook-lab`.
2. Заархивируйте именно папку навыка.
3. В Claude откройте `Settings` → `Customize` → `Skills`.
4. Создайте навык и загрузите один архив.

Claude.ai принимает навыки по одному. Для полного набора загрузите пять архивов.

### Claude Code

Проектная установка:

```bash
cp -R hook-lab .claude/skills/hook-lab
```

Личная установка:

```bash
mkdir -p ~/.claude/skills
cp -R hook-lab ~/.claude/skills/hook-lab
```

### Codex

Проектная установка:

```bash
cp -R hook-lab .codex/skills/hook-lab
```

Личная установка:

```bash
mkdir -p ~/.codex/skills
cp -R hook-lab ~/.codex/skills/hook-lab
```

### OpenCode

Поддерживаемые проектные пути:

```text
.opencode/skills/<skill-name>/SKILL.md
.claude/skills/<skill-name>/SKILL.md
.agents/skills/<skill-name>/SKILL.md
```

Поддерживаемые личные пути:

```text
~/.config/opencode/skills/<skill-name>/SKILL.md
~/.claude/skills/<skill-name>/SKILL.md
~/.agents/skills/<skill-name>/SKILL.md
```

Подробнее: [установка и совместимость](docs/installation.md).

## Что происходит внутри каждого навыка

### 1. `voice-print` фиксирует наблюдаемый голос

Он читает образцы и собирает лексический отпечаток, длину фраз, привычки пунктуации, регистр, отношение к читателю, структуру и список запретов. Профиль содержит примеры наблюдений и тестовую фразу. При трёх образцах результат помечается как предварительный. При смешанных жанрах навык описывает диапазон каждого режима письма.

### 2. `audience-dig` отделяет сигнал от шума

Он проходит по каждой реплике, исключает спам и реакции без содержания, затем ранжирует фразы по повторяемости и эмоциональной силе. В отчёт попадают вопросы, проблемы, конкретные похвалы и готовые формулировки для будущего контента. Личные данные не переносятся в финальный ответ.

### 3. `hook-lab` выбирает рабочее напряжение первой строки

В библиотеке есть семь формул: зеркало боли, открытие желания, разрыв любопытства, контринтуитивная мысль, предъявление результата, было/стало и обрыв паттерна. Формула не подставляется принудительно. Если она слаба для темы, навык объясняет это и даёт более подходящий ход. Конкретные цифры берутся только из подтверждённых данных.

### 4. `script-smith` считает время, а не только слова

Сценарий собирается из четырёх блоков: хук, доверие, тело и CTA. Для 15 секунд мысль сжимается до одного хода. В 30 секундах помещается классическая схема. В ролике на 45–60 секунд можно раскрыть два или три шага. После 90 секунд появляются мини-хуки, чтобы удерживать внимание по ходу рассказа.

### 5. `repurpose` меняет структуру под формат

Он сначала извлекает центральную мысль из исходника. Потом пересобирает её под законы конкретной площадки. У видео есть темп и монтажные склейки. У карусели один тезис на слайд. У письма больше воздуха для контекста. У треда отдельная мысль на каждый пост. Этот шаг защищает материал от косметической замены переносов строк.

Для полного справочника откройте [навыки и их контракты](docs/skill-reference.md).

## Ограничения и аккуратность

- Не вставляйте личные данные клиентов, номера телефонов, адреса, закрытые ссылки и ключи доступа.
- Комментарии конкурентов можно анализировать как источник паттернов. Формулировки и материалы не следует переносить дословно в свой контент.
- `voice-print` предназначен для собственного голоса, команды с разрешением или явно согласованного бренд-стиля. Он не предназначен для имитации публичного человека по нескольким текстам.
- Любая статистика, дата, цитата и результат в готовом контенте требуют проверки до публикации.
- Навыки готовят черновики и структуру. Решение о публикации остаётся за человеком, который знает факты, контекст и обязательства перед аудиторией.

Подробности: [приватность и этика](docs/privacy-and-ethics.md).

## Вклад и развитие

Перед добавлением нового навыка проверьте четыре вещи:

1. У него есть конкретный вход, результат и условие запуска.
2. Он не дублирует существующий этап без новой функции.
3. У него есть правила для малого объёма данных, неоднозначности и приватных материалов.
4. В `SKILL.md` есть границы: чего навык не делает и какие допущения обязан назвать.

Новые идеи и исправления оформляйте через issue или pull request. Не прикладывайте в issue закрытые переписки, персональные данные и токены. Полный порядок описан в [CONTRIBUTING.md](CONTRIBUTING.md), правила раскрытия уязвимостей — в [SECURITY.md](SECURITY.md).
